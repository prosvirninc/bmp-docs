# Окупаемость SuperFix

## Входные данные

| Источник | Описание |
|----------|----------|
| PBA | Плановый GRP"20 с листа "Свод" |
| PBA | Плановый GRP"20 с листа "Свод сток" |
| База данных | Среднерыночная доля SuperFix по телеканалам |
| Mediascope | Выгрузка по кампании: TVR, GRP, Prime и Off Prime, Дата, Телеканал |
| Mediascope | Affinity и TVR по рынку |

---

## Стандартный расчет без учета стока

Применяется, если на листе «Свод сток» для данной кампании **нет записей**.

### Доля SuperFix по каналам

Для каждого телеканала:

```
SuperFix = ('План GRP"20 / мин SFix PT' + 'План GRP"20 / мин SFix OP') / 'План GRP"20 / мин ВСЕГО'
```

> `План GRP"20 / мин ВСЕГО` включает все типы закупки: Максимум + SuperFix PT + SuperFix OP + Фикс.

---

### Affinity кампании

Для каждого телеканала:

```
TRP кампании  = Σ Big TVR {ЦА}     — Сумма рейтингов роликов на ЦА
GRP кампании  = Σ Big Sales TVR    — Сумма рейтингов роликов на БА

Affinity кампании = 'TRP кампании' / 'GRP кампании'
```

---

### Affinity по рынку

Запрос к Mediascope за период кампании:

```
Affinity рынка = 'TRP рынка' / 'GRP рынка'   (агрегировано по телеканалу)
```

---

### Индекс Affinity

```
Индекс Affinity = ('Affinity кампании' / 'Affinity рынка') × 100
```

---

### Порог окупаемости

```
Порог = (1.0 + (SuperFix − 'SuperFix ср.') × 0.15) × 100
```

Где `SuperFix ср.` — средняя доля SuperFix для данного телеканала и года. Уже задано, не вычисляется.

---

### Prime-индекс

Из выгрузки определяется средний рейтинг в Prime Time (PT) и Off-Prime (OP):

```
TVR PT ср. = Σ TVR PT / кол-во роликов в PT
TVR OP ср. = Σ TVR OP / кол-во роликов в OP
```

Рыночные значения — из Mediascope API (4 сегмента: PT в будни и выходные, OP в будни и выходные):

```
Индекс PT = (TVR PT ср. / TVR PT ср. рынка) × 100
Индекс OP = (TVR OP ср. / TVR OP ср. рынка) × 100
```

---

### Индекс эффективности постановки

Весовые коэффициенты зависят от рекламодателя:

| Рекламодатель | Вес AFF | Вес PT | Вес OP |
|---------------|-------|------|------|
| Стандарт | 0.7 | 0.2 | 0.1 |
| Особый случай | 0.5 | 0.3 | 0.2 |

```
Индекс Эффективности = Индекс Affinity × Вес AFF + Индекс PT × Вес PT + Индекс OP × Вес OP
```

Граничные случаи (нормировка на сумму присутствующих весов):

- Если `Индекс OP = 0`: `Индекс Эффективности = (Индекс Affinity × Вес AFF + Индекс PT × Вес PT) / (Вес AFF + Вес PT)`
- Если `Индекс PT = 0`: `Индекс Эффективности = (Индекс Affinity × Вес AFF + Индекс OP × Вес OP) / (Вес AFF + Вес OP)`

---

### Итоговый индекс окупаемости SuperFix

```
Индекс SuperFix = (Индекс Эффективности / Порог) × 100
```

Значение `> 100` означает, что кампания «окупает» SuperFix-надбавку.

---

### Строка «Итого»

Взвешенные суммы по долям GRP каждого канала:

```
GRP вес = GRP канал / GRP всего

SuperFix итого      = Σ (SuperFix × GRP вес)
Affinity кампании итого = Σ (Affinity кампании × GRP вес)
Affinity рынка итого = Σ (Affinity рынка × GRP вес)
SuperFix ср. итого  = Σ (SuperFix ср. × GRP вес)

Индекс Affinity итого = (Affinity кампании итого / Affinity рынка итого) × 100
Порок итого      = (1 + (SuperFix итого − SuperFix ср. итого) × 0.15) × 100

Индекс Prime «Итого» взвешивается по количеству роликов (PT u. / OP u.)
```

## Сценарий 2 — С учётом стока

Применяется, если в «свод сток» для данной кампании **есть строки**.  
Расчёт аффинити, Prime-индекса и индекса эффективности — **без изменений**.

> **Частный случай — 100% сток.** Если кампания присутствует только в «свод сток»
> и отсутствует в «свод», расчёт всё равно выполняется: `GRP_regular = 0`,
> `SuperFix = 0`, `% stock = 1.0`. Affinity и Prime-индекс считаются по выгрузке
> как обычно, а формула порога автоматически даёт `Порог = (1 − 0.15×SF_avg − (1−coef)) × 100`.

### Дополнительные данные из листа «свод сток»

- `Plan_GRP_ВСЕГО_stock` — плановый GRP"20 стока по телеканалу
- `stock_coef` — коэффициент «Кэф к обычном (сток / обычное)»:
  - индивидуален для каждого клиента и телеканала
  - вносится вручную в лист «свод сток»
  - **допустимый диапазон: от 0.1 до 1.0** (проверяется при загрузке)
  - должен быть одинаковым для всех строк одной пары (кампания / канал)

---

### Шаг 1 (изменён). SuperFix-доля и доля стока по каналу

```
GRP_regular   = Plan_GRP_ВСЕГО         (из листа «свод»)
GRP_stock     = Plan_GRP_ВСЕГО_stock   (из листа «свод сток»)
GRP_total_all = GRP_regular + GRP_stock

SuperFix  = (Plan_GRP_SFix_PT + Plan_GRP_SFix_OP) / GRP_total_all
% stock   = GRP_stock / GRP_total_all
```

Числитель SuperFix не меняется — SFix-GRP берётся только из обычного листа.

---

### Шаг 5 (изменён). Порог с учётом стока

Целевой % стока по рынку = **0** (сток — сверх нормы).

```
Порог = (1 + 0.15 × (SuperFix − SuperFix_avg) − % stock × (1 − stock_coef)) × 100
```

**Пример** при `stock_coef = 0.8`:

```
Порог = (1 + 0.15 × (SuperFix − SuperFix_avg) − % stock × 0.2) × 100
```

Чем больше доля стока — тем ниже порог (сток дешевле обычного размещения).

---

### Строка «Итого» со стоком

`% stock` и `stock_coef` взвешиваются по GRP SOV:

```
% stock_итого   = Σ (% stock × GRP_SOV)
coef_итого      = Σ (stock_coef × GRP_SOV)

Threshold_итого = (1 + 0.15 × (SuperFix_итого − SuperFix_avg_итого)
                   − % stock_итого × (1 − coef_итого)) × 100
```

---

## Присутствие стока в выводе

| Ситуация | Поведение |
|----------|-----------|
| В «свод сток» нет строк по кампании | Стандартный расчёт; колонки «% стока» и «Кэф стока» в Excel **не появляются** |
| В «свод сток» есть строки по кампании | Расчёт со стоком; колонки «% стока» и «Кэф стока» **добавляются** в лист «Индекс Affinity» |
