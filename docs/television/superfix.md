# SuperFix

## Входные данные

| Источник | Описание |
|----------|----------|
| PBA | Плановый GRP"20 с листа "Свод" |
| PBA | Плановый GRP"20 с листа "Свод сток" |
| База данных | Среднерыночная доля SuperFix по телеканалам |
| Palomars | Основная выгрузка |
| Mediascope | Affinity и TVR по рынку |

**Используемые столбцы:** TVR, GRP, Prime и Off Prime, Дата, Телеканал

---

## Стандартный расчет (без учета стока)

Применяется, если на листе «Свод сток» для анализируемой кампании **нет записей**.

### Доля SuperFix по каналам

Для каждого телеканала:

```
'SuperFix' = ('План GRP"20 / мин SFix PT' + 'План GRP"20 / мин SFix OP') / 'План GRP"20 / мин ВСЕГО'
```

> `'План GRP"20 / мин ВСЕГО'` включает все типы закупки: Максимум + SuperFix PT + SuperFix OP + Фикс.

---

### Affinity кампании

Для каждого телеканала по данным выгрузки:

```
'TRP камп.' = Σ Big TVR {ЦА}   — сумма рейтингов роликов на целевую аудиторию
'GRP камп.' = Σ Big Sales TVR  — сумма рейтингов роликов на баинговую аудиторию

'Affinity камп.' = 'TRP камп.' / 'GRP камп.'
```

---

### Affinity по рынку

Запрос к Mediascope за период кампании, агрегировано по телеканалу:

```
'Affinity рын.' = 'TRP рын.' / 'GRP рын.'
```

---

### Индекс Affinity

```
'Индекс Affinity' = ('Affinity камп.' / 'Affinity рын.') × 100
```

---

### Порог окупаемости

```
'Порог' = (1 + 0.15 × ('SuperFix' − 'SuperFix ср.')) × 100
```

`'SuperFix ср.'` — среднерыночная доля SuperFix для указанного телеканала и года. Задается из базы данных, не вычисляется.

---

### Prime-индекс

Из выгрузки — средний рейтинг в Prime Time (PT) и Off‑Prime (OP):

```
'TVR PT ср.' = Σ TVR PT / кол-во роликов в PT
'TVR OP ср.' = Σ TVR OP / кол-во роликов в OP
```

Рыночные значения — из Mediascope API (4 сегмента: PT в будни и выходные, OP в будни и выходные):

```
'Индекс PT' = ('TVR PT ср.' / 'TVR PT ср. рын.') × 100
'Индекс OP' = ('TVR OP ср.' / 'TVR OP ср. рын.') × 100
```

---

### Индекс эффективности постановки

Весовые коэффициенты зависят от рекламодателя:

| Рекламодатель | 'Вес&nbsp;AFF' | 'Вес&nbsp;PT' | 'Вес&nbsp;OP' |
|---------------|:---:|:---:|:---:|
| Стандарт | 0.7 | 0.2 | 0.1 |
| Особый случай | 0.5 | 0.3 | 0.2 |

```
'Индекс Эффективности' = 'Индекс Affinity' × 'Вес AFF' + 'Индекс PT' × 'Вес PT' + 'Индекс OP' × 'Вес OP'
```

Граничные случаи — нормировка на сумму присутствующих весов:

- Если 'Индекс OP' = 0:  
  `'Индекс Эффективности' = ('Индекс Affinity' × 'Вес AFF' + 'Индекс PT' × 'Вес PT') / ('Вес AFF' + 'Вес PT')`
- Если 'Индекс PT' = 0:  
  `'Индекс Эффективности' = ('Индекс Affinity' × 'Вес AFF' + 'Индекс OP' × 'Вес OP') / ('Вес AFF' + 'Вес OP')`

---

### Итоговый индекс окупаемости SuperFix

```
'Индекс SuperFix' = ('Индекс Эффективности' / 'Порог') × 100
```

Значение `> 100` означает, что кампания окупает SuperFix размещение.

---

### Итоговые значения

Взвешивание по доле GRP каждого канала в общем объеме:

```
'GRP вес' = 'GRP канал' / 'GRP всего'

'SuperFix' = Σ ('SuperFix' × 'GRP вес')
'Affinity камп.' = Σ ('Affinity камп.' × 'GRP вес')
'Affinity рын.' = Σ ('Affinity рын.' × 'GRP вес')
'SuperFix ср.' = Σ ('SuperFix ср.' × 'GRP вес')

'Индекс Affinity' = ('Affinity камп.' / 'Affinity рын.') × 100
'Порог' = (1 + 0.15 × ('SuperFix' − 'SuperFix ср.')) × 100
```

Индексы PT и OP взвешиваются по количеству роликов (не по GRP).

---

## Расчет с учетом стока

Применяется, если на листе «Свод сток» для анализируемой кампании **есть записи**.

Расчет отличается от стандартного в двух местах: **доля SuperFix** (шаг 1) и **порог окупаемости** (шаг 5). Остальные шаги — Affinity кампании, Affinity по рынку, Индекс Affinity, Prime‑индекс и Индекс эффективности — идентичны стандартному расчету.

> **Частный случай — 100% сток.** Если кампания присутствует только в «Свод сток» и отсутствует в «Свод», расчет все равно выполняется: 'GRP обычн.' = 0, 'SuperFix' = 0, 'Доля стока' = 100%. Affinity и Prime‑индекс считаются по выгрузке как обычно. Формула порога автоматически упрощается до `(1 − 0.15 × 'SuperFix ср.' − (1 − 'K стока')) × 100`.

### Дополнительные данные из «Свод сток»

Для каждой пары (кампания / телеканал) вносятся вручную два поля:

- **`'GRP сток'`** (`'План GRP"20 / мин ВСЕГО (сток)'`) — плановый объем GRP"20 стока по каналу.
- **`'K стока'`** (`K` — коэффициент) — ценовой коэффициент стока относительно обычного размещения. Индивидуален для каждого клиента и телеканала. Допустимый диапазон: **0.1–1.0** (проверяется при загрузке). Должен совпадать во всех строках одной пары (кампания / канал).

---

### Доля SuperFix по каналам

Знаменатель расширяется на объем стока; числитель остается прежним — SFix‑GRP берется только из листа «Свод»:

```
'GRP обычн.' = 'План GRP"20 / мин ВСЕГО'          (лист «Свод»)
'GRP сток' = 'План GRP"20 / мин ВСЕГО (сток)'     (лист «Свод сток»)
'GRP всего' = 'GRP обычн.' + 'GRP сток'

'SuperFix' = ('План GRP"20 / мин SFix PT' + 'План GRP"20 / мин SFix OP') / 'GRP всего'
'Доля стока'  = 'GRP сток' / 'GRP всего'
```

---

### Affinity кампании

Считается так же, как в стандартном расчете:

```
'TRP камп.' = Σ Big TVR {ЦА}   — сумма рейтингов роликов на целевую аудиторию
'GRP камп.' = Σ Big Sales TVR  — сумма рейтингов роликов на баинговую аудиторию

'Affinity камп.' = 'TRP камп.' / 'GRP камп.'
```

---

### Affinity по рынку

Считается так же, как в стандартном расчете:

```
'Affinity рын.' = 'TRP рын.' / 'GRP рын.'   (агрегировано по телеканалу)
```

---

### Индекс Affinity

```
'Индекс Affinity' = ('Affinity камп.' / 'Affinity рын.') × 100
```

---

### Порог окупаемости

Чем выше доля стока, тем ниже порог: стоковое размещение дешевле обычного, поэтому от кампании требуется меньший показатель для окупаемости размещения. Целевая доля стока по рынку равна нулю, поэтому 'Доля стока' входит в формулу целиком.

```
'Порог' = (1 + 0.15 × ('SuperFix' − 'SuperFix ср.') − 'Доля стока' × (1 − 'K стока')) × 100
```

**Пример** при `'K стока' = 0.8`:

```
'Порог' = (1 + 0.15 × ('SuperFix' − 'SuperFix ср.') − 'Доля стока' × 0.2) × 100
```

---

### Prime-индекс

Считается так же, как в стандартном расчете:

```
'TVR PT ср.' = Σ TVR PT / кол-во роликов в PT
'TVR OP ср.' = Σ TVR OP / кол-во роликов в OP
```

```
'Индекс PT' = ('TVR PT ср.' / 'TVR PT ср. рын.') × 100
'Индекс OP' = ('TVR OP ср.' / 'TVR OP ср. рын.') × 100
```

---

### Индекс эффективности постановки

Считается так же, как в стандартном расчете (веса — по таблице выше):

```
'Индекс Эффективности' = 'Индекс Affinity' × 'Вес AFF' + 'Индекс PT' × 'Вес PT' + 'Индекс OP' × 'Вес OP'
```

---

### Итоговый индекс окупаемости SuperFix

```
'Индекс SuperFix' = ('Индекс Эффективности' / 'Порог') × 100
```

Значение `> 100` означает, что кампания окупает SuperFix размещение.

---

### Итоговые значения

Взвешивание по доле GRP каждого канала. К стандартным полям добавляются 'Доля стока' и 'K стока':

```
'GRP вес' = 'GRP канал' / 'GRP всего'

'SuperFix' = Σ ('SuperFix' × 'GRP вес')
'SuperFix ср.' = Σ ('SuperFix ср.' × 'GRP вес')
'Доля стока' = Σ ('Доля стока' × 'GRP вес')
'K стока' = Σ ('K стока' × 'GRP вес')
'Affinity камп.' = Σ ('Affinity камп.' × 'GRP вес')
'Affinity рын.' = Σ ('Affinity рын.' × 'GRP вес')

'Индекс Affinity' = ('Affinity камп.' / 'Affinity рын.') × 100
'Порог' = (1 + 0.15 × ('SuperFix' − 'SuperFix ср.') − 'Доля стока' × (1 − 'K стока')) × 100
```

Индексы PT и OP взвешиваются по количеству роликов (не по GRP).
